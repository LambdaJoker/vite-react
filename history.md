# 项目历史与关键决策记录

该文件用于跟踪在此项目开发期间做出的重要架构更改、性能优化、阻塞性问题及其解决方案和相关决策。（⚠️ 注意：本文件仅保留最近/最重要的 10 条历史记录）

### 1. 2026-04-12 - 完善文章功能：标签过滤与时间轴归档
- **目标:** 提升用户阅读体验，增加文章分类浏览和归档展示功能。
- **操作:** 在 ArticleDetail 底部增加“上一篇/下一篇”导航；在 ArticleList 增加基于 useMemo 的标签过滤功能；新建 Archive 页面以“年份-月份”分组展示时间轴。
- **结果:** 增强了博客系统的导航性，用户可以更方便地按标签或时间线浏览文章。

### 2. 2026-04-12 - 模块级重构与性能优化
- **目标:** 提升应用响应速度，优化前端组件代码可读性。
- **操作:** 后端引入 `compression` 中间件优化 JSON 响应体积；前端将 ArticleList 内部的列表项抽离为独立的 ArticleCard 组件，提升代码可读性。
- **结果:** 数据传输速度更快，前端代码结构更清晰。

### 3. 2026-04-12 - 优化项目展示模块 (Projects Showcase)
- **目标:** 改进 `/projects` 路由下的视觉呈现和代码模块化。
- **操作:** 将项目列表中的卡片 UI 抽离为独立的 ProjectCard 组件；重构搜索、分类和标签筛选区域，统一采用玻璃拟态 (Glassmorphism) 容器设计。
- **结果:** 项目展示页面的层级更清晰，UI 更现代化，筛选逻辑更易维护。

### 4. 2026-04-12 - 增加图床与 Markdown 图片拖拽上传功能
- **目标:** 改善撰写文章时的体验，允许用户直接在编辑器中上传图片至 Vercel Blob。
- **操作:** 后端新增 `/upload-image` 接口对接 `@vercel/blob` SDK；前端 `ReactMde` 编辑器实现 `paste` 和拖拽事件拦截，自动上传文件并回填 Markdown 图片语法。
- **结果:** 博客系统现具备完整的图床功能，大幅简化了文章配图流程。

### 5. 2026-04-12 - 修复 Vercel Blob 绝对路径图片预览 Bug
- **目标:** 确保管理员在编辑文章时，已上传至 Vercel Blob 的图片能正确在封面图预览中显示。
- **操作:** 修复了 `ArticleForm/index.tsx` 中由于强制拼接 `VITE_API_BASE_URL` 导致绝对路径图片失效的 Bug，统一使用 `getImageUrl` 助手函数处理路径。
- **结果:** 历史数据中存储的本地相对路径图片和新上传的 Vercel Blob 绝对路径图片均能正常预览。

### 6. 2026-04-12 - 修复 Vercel 生产环境下 TOC 目录不显示问题
- **目标:** 修复在生产环境或常见笔记本屏幕尺寸下，文章详情页的目录 (TOC) 无法显示的问题。
- **操作:** 修改了 `ArticleDetail/index.css` 的媒体查询，将完全隐藏 TOC 的阈值从 `1500px` 降低到 `1200px`；在 `1200px` 到 `1500px` 的屏幕宽度下，将 TOC 改为固定在右侧边缘 (`right: 20px`)，避免遮挡正文同时保证可见性。同时优化了 `ArticleDetail/index.tsx` 中提取标题的正则表达式为 `/^(##|###)\s+([^\r\n]+)/gm`，避免不同操作系统下换行符 `\r` 导致的锚点链接失效。
- **结果:** 在常见尺寸的笔记本屏幕上（如 1366px, 1440px）能够正常显示目录，且保证了生产环境跨平台换行符的兼容性。

### 7. 2026-04-12 - 全局替换作者信息及应用名称为 "Random Glow"
- **目标:** 将整个项目中所有的旧名字（如 "TAO"、"taotao"、"Admin"）统一更新为用户的名字 "Random Glow"。
- **操作:** 
  1. 编写 Node 脚本全局替换了所有组件头部注释的 `@Author` 和 `@LastEditors`。
  2. 替换了前端 UI 中硬编码的名称，如 `Header` 的 title 属性和 `<title>` 标签。
  3. 修改了后端控制器及初始化 SQL 中的默认 `author` 值为 "Random Glow"。
  4. 修改了 Prisma Schema 的默认值，并执行 `prisma generate` 和 `prisma db push`。
  5. 编写并运行了针对 Prisma 数据库的脚本，更新了现有文章和评论表中的作者信息。
- **结果:** 整个代码库及数据库中的作者署名和应用名称都已成功切换为 "Random Glow"。

### 8. 2026-04-12 - 修复 Header 标题换行问题
- **目标:** 修复页面顶部导航栏中，由于较长的标题 ("Random Glow") 导致文字换行的问题。
- **操作:** 在 `Header/index.css` 中为 `.logo` 样式类添加了 `white-space: nowrap` 属性，并清理了可能会引发宽度冲突的过时 `.logo` 和 `.logo img` 样式。
- **结果:** 标题现在能够完整且在一行内显示，排版更加美观。

### 9. 2026-04-12 - 修复文章详情页评论内容居中对齐问题
- **目标:** 修复文章详情页的评论内容因继承全局样式而意外居中对齐的问题。
- **背景:** 根组件 `#root` 设置了 `text-align: center`，导致评论内容 `<MarkdownRenderer>` 渲染出的段落文字全部居中显示，与左对齐的头像和昵称在视觉上非常不协调。
- **操作:** 在 `ArticleDetail/index.css` 中，为 `.comment-content` 及其所有后代元素（`*`）强制添加了 `text-align: left;` 样式属性。
- **结果:** 评论的具体内容现已正确向左对齐，与头像、昵称和底部操作栏对齐方式保持一致。

### 10. 2026-04-12 - 优化文章详情页主内容宽度与目录（TOC）排版
- **目标:** 缩小文章主内容区域的宽度，使其阅读体验更紧凑舒适，同时修复目录栏被挤压的问题。
- **操作:**
  1. 将 `.article-detail-container` 的 `max-width` 从 `900px` 减小至 `800px`。
  2. 调整悬浮目录 `.table-of-contents` 的相对定位，将其 `left` 从 `calc(50% + 480px)` 更新为 `calc(50% + 430px)`，以匹配缩小后的主内容宽度。
  3. 修改了媒体查询中的样式，在屏幕宽度介于 `1200px` 和 `1500px` 时，将目录宽度稍微缩小为 `200px` 以避免相互重叠。
  4. 为目录栏 `.table-of-contents` 增加了隐藏滚动条的跨浏览器 CSS 代码。
- **结果:** 文章主内容的视觉宽度更加聚拢，右侧目录也能与正文保持合适的间距，整体排版更加精致。
