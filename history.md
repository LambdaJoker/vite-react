# 项目历史与关键决策记录

该文件用于跟踪在此项目开发期间做出的重要架构更改、性能优化、阻塞性问题及其解决方案和相关决策。（⚠️ 注意：本文件仅保留最近/最重要的 10 条历史记录）

### 22. 2026-04-20 - 首页 (Home) UI 组件升级与 CTA 按钮优化
- **目标:** 利用 `ui-ux-pro-max` 技能对首页的 Hero 区域及按钮进行视觉优化，提高转化率 (CTA) 体验。
- **操作:** 
  1. **Hero 卡片质感:** 升级了 `.hero-section` 的背景，应用了全局一致的更强烈的毛玻璃参数 (`blur(12px)`) 和深层阴影 (`0 8px 32px rgba(0, 0, 0, 0.08)`)，并将边框圆角柔化为 `24px`。
  2. **主行动按钮 (Primary CTA):** 重构了主按钮的视觉层，将生硬的纯色块替换为带动态阴影的品牌渐变色 (`linear-gradient(135deg, #007bff, #00ff88)`)，采用全圆角 (`30px`) 胶囊设计，并增加了字重 (`600`) 以吸引点击。
  3. **次行动按钮 (Secondary CTA):** 调整为半透明毛玻璃材质 (`rgba(255, 255, 255, 0.5)`)，弱化边框并在悬浮时增强不透明度，与主按钮形成强烈的层级对比。
  4. **悬浮微动效:** 将按钮的交互反馈替换为定制贝塞尔曲线 (`cubic-bezier(0.4, 0, 0.2, 1)`)，悬浮时 `translateY(-3px)` 配合动态扩散的发光阴影。
- **结果:** 首页第一屏 (Above the fold) 视觉重心更加明确，现代感的渐变和动效极大提升了按钮的点击欲望 (Click-through Rate)。

### 23. 2026-04-20 - 网站页脚 (Footer) 质感升级与细节微调
- **目标:** 使网站底部的 Footer 区域与经过优化的主视觉保持一致的质感与层级感。
- **操作:** 
  1. 提升了 `.app-footer` 的毛玻璃模糊度 (`blur(12px)`) 和背景不透明度 (`rgba(255, 255, 255, 0.4)`)，并稍微增强了顶边框的亮度，让页脚与背景视频融合得更好且文字更清晰。
  2. 将 Footer 链接的间距拉宽 (`gap: 2rem`)，增加了文字大小和颜色的对比度，并为其添加了鼠标悬浮时的轻微上浮效果 (`translateY(-2px)`) 和颜色渐变过渡。
- **结果:** 页脚不再显得过于单薄或被背景掩盖，整个站点的头尾设计语言实现了完美闭环。

### 24. 2026-04-20 - 全局组件易用性与视觉一致性提升
- **目标:** 利用 `ui-ux-pro-max` 规范，对页面边角组件 (如“回到顶部”按钮) 进行触控和视觉升级。
- **操作:** 
  1. 将悬浮的“回到顶部”按钮 (`.scroll-top-button-reusable`) 的尺寸从 `48px` 增加到 `52px`，满足移动端无障碍触控目标 (Touch Target) 大于 `44px` 的最佳实践。
  2. 同步了毛玻璃参数 (`blur(12px)`) 和阴影。
  3. 更改了悬停状态 (Hover) 的样式，移除了死板的纯色块，换成了全局统一的品牌渐变色 (`linear-gradient(135deg, #007bff, #00ff88)`) 和动态阴影 (`translateY(-6px)`)。
- **结果:** 全站所有按钮组件 (CTA 按钮、回到顶部等) 的反馈语言保持高度一致，提高了可访问性。

### 25. 2026-04-20 - 子页面 (文章、项目) UI 容器重构与细节优化
- **目标:** 利用 `ui-ux-pro-max` 将全站深层页面的卡片和交互状态统一为现代化的设计系统标准。
- **操作:** 
  1. **文章/项目列表容器:** 统一 `ArticleList` 和 `ProjectsContent` 顶部标题容器的毛玻璃参数，将其圆角拉伸到 `24px`，并加入更柔和的环境阴影 (`box-shadow: 0 8px 32px rgba(0,0,0,0.08)`)。
  2. **文章详情页:** 优化了正文阅读区域的背景卡片，同样的圆角与深层阴影处理，大幅提升了长图文阅读时的沉浸感。
  3. **互动组件:** 重构了文章详情底部的“标签(Tag)”和“分享(Share)”按钮。标签加入了微弱的边框和悬浮阴影；分享按钮替换为全站统一的主色渐变 (`#007bff` to `#00ff88`) 和弹性微动效 (`translateY(-3px)`)。
- **结果:** 整个项目的组件设计语言实现了从首页到子页面、从容器到按钮的完全统一，消除了突兀的视觉断层。

### 26. 2026-04-20 - 网站输入框与表单元素 UI 统一与质感优化
- **目标:** 使用 `ui-ux-pro-max` 将输入框 (Input)、文本域 (Textarea) 等表单元素升级为现代化的毛玻璃风格，统一焦点状态。
- **操作:** 
  1. 移除了默认的死板白色背景，应用了带透明度 (`rgba(255, 255, 255, 0.4)`) 和环境模糊 (`backdrop-filter: blur(8px)`) 的设计，使输入框自然融入动态背景。
  2. 加入了细微的白边 (`border: 1px solid rgba(255, 255, 255, 0.5)`) 和内阴影以增加立体感。
  3. 增强了 Focus 状态的交互反馈：激活时放大边框颜色 (`rgba(0, 123, 255, 0.5)`)，增强背景透明度 (`0.6`) 并加入平滑的发光效果 (`box-shadow`)。
- **结果:** 整个站点的搜索框、留言板、登录框都获得了与卡片设计语言高度一致的次世代玻璃质感。

### 27. 2026-04-20 - 骨架屏加载动画优化
- **目标:** 使网络请求或页面懒加载时的过渡骨架屏 (Skeleton Loader) 符合玻璃拟态的整体风格。
- **操作:** 
  1. 为 `.skeleton` 容器加入了带透明度的边框和背景 (`rgba(255, 255, 255, 0.4)`)，并统一了毛玻璃效果 (`blur(8px)`)。
  2. 优化了 `shimmer` 骨架屏扫光动画的时间线，从线性的 `1.5s` 改为带有呼吸节奏的 `1.8s cubic-bezier(0.4, 0, 0.2, 1)`。
- **结果:** 在弱网环境下，页面的加载等待过程从刺眼的灰色方块变成了具有光影质感的透明占位符，有效降低了用户的等待焦虑。

### 28. 2026-04-20 - 修复移动端视频背景黑屏及 UI 适配问题
- **目标:** 修复手机端点击壁纸时出现白色高亮背景的问题。
- **操作:** 
  1. 在 `DynamicBackground/index.tsx` 的 `<video>` 标签中添加了 `style={{ backgroundColor: 'transparent' }}`，防止视频加载期间由于 `poster` 透明度问题或默认背景色导致的白色闪烁。
  2. 在 `index.css` 的 `:root` 全局样式中添加了 `-webkit-tap-highlight-color: transparent;`，彻底禁用了移动端浏览器（如 Safari 和微信 X5）在点击任意区域时默认产生的半透明白色或灰色点击高亮块。
- **结果:** 手机端点击页面空白处（壁纸）时不再出现不美观的白色闪烁块，提升了纯净的沉浸式体验。
