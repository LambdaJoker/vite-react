# 🚀 博客项目全栈部署指南 (Vercel 免费版)

本项目已经过彻底改造，完全兼容 **Vercel Serverless** 环境。你可以把前端和后端一起在 Vercel 上实现**一劳永逸的“一键双杀”免费部署**，全程绝对不需要绑定任何信用卡。

## ⚠️ 注意事项与 Serverless 限制
由于 Vercel 运行的是 Serverless (无服务器函数)，相比于传统的长期运行的服务器容器（如 Render）：
1. **图片上传限制**：Vercel 的硬盘是只读的，项目中的 `multer` 已自动降级为内存存储，不支持将图片长期保存到本地。如需永久保存图片，请接入第三方图床（如 Cloudinary、OSS）。
2. **后台定时任务改为按需触发**：壁纸爬虫原本使用 `setInterval` 运行，现已改为懒加载（当 Vercel 环境下没有缓存且用户请求壁纸时，实时爬取并写入 `/tmp` 缓存）。

## 🛠️ Vercel 部署步骤

### 第一步：提交代码到 GitHub
确保你本地的最新代码都已经 commit 并 push 到了 GitHub 仓库。
```bash
git add .
git commit -m "feat: 适配 Vercel 全栈部署"
git push origin main
```

### 第二步：在 Vercel 导入项目
1. 登录 [Vercel Dashboard](https://vercel.com/dashboard)。
2. 点击右侧黑色的 **"Add New..." -> "Project"**。
3. 在 "Import Git Repository" 列表中，找到你的 GitHub 仓库并点击 **Import**。

### 第三步：配置部署参数 (❗关键)
在配置页面，请严格按照以下说明展开并填写：

- **Framework Preset**: 保持默认（Vite 或 Other 均可）。
- **Root Directory (根目录)**: 
  - 点击 `Edit`，选择 **`vite-react`** （因为代码主要在此目录下）。
- **Build Command (构建命令)**: 
  - 保持默认（Vercel 会自动识别到 `package.json` 中的 `vercel-build`）。
- **Output Directory (输出目录)**: 
  - 必须手动覆盖填写为 **`packages/client/dist`** （这是前端 Vite 打包后的产物位置）。

### 第四步：配置环境变量
点击 **"Environment Variables"** 展开面板，添加你的数据库连接（必须）：

| Name | Value |
| :--- | :--- |
| `DATABASE_URL` | `mysql://avnadmin:你的密码@你的Aiven数据库地址.aivencloud.com:端口/defaultdb?ssl-mode=REQUIRED` |

> ⚠️ **注意**：使用 Aiven MySQL 必须在链接末尾带上 `?ssl-mode=REQUIRED` 以确保安全连接。

### 第五步：一键部署
点击下方的 **Deploy** 按钮。
Vercel 会自动执行以下流程：
1. 运行 `postinstall` 脚本安装并生成 Prisma Client。
2. 运行 `vercel-build` 打包前端 Vite 项目。
3. 识别根目录的 `vercel.json` 并将 `api/index.ts` 转换为 Serverless API 函数。

等待 1-2 分钟，部署完成后，你的前端和后端将运行在同一个 Vercel 免费域名下（如 `https://your-project.vercel.app`），前端请求 `/api/...` 会自动由后端 Serverless 函数处理。

## 💡 本地开发说明
项目已配置好完整的本地开发脚本：
- 启动完整项目 (前端 + 后端)：
  ```bash
  npm run dev:full
  ```
- 单独启动前端：
  ```bash
  npm run dev
  ```
- 单独启动后端：
  ```bash
  npm run dev:server
  ```
